## 一、系统概述

本项目是一个基于 **RAG（检索增强生成）** 技术的智能问答系统，专为宁波工程学院网络空间安全学院设计。系统使用大语言模型（LLM）作为基座，结合学院专属知识库，为用户提供准确、可靠的学院信息问答服务。

**核心特性：**

- 基于 LangChain + LangGraph 构建 RAG 问答流程
- 支持 DeepSeek API 和 Ollama 本地模型两种 LLM 提供商
- 向量知识库（ChromaDB）存储学院各类信息
- 流式输出（SSE），逐 token 显示回答
- 多用户支持，会话独立管理
- 管理员控制台，完整管理后台
- 系统审计日志，记录所有用户操作

## 三、快速启动

### 第一步：配置环境变量

复制环境变量模板并编辑：

```bash
cp .env.example .env
```

编辑 `.env` 文件，至少配置以下内容之一：

**选项 A：使用 DeepSeek API（推荐）**

```ini
LLM_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-your-api-key-here
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_BASE_URL=https://api.deepseek.com
```

**选项 B：使用 Ollama 本地模型**

```ini
LLM_PROVIDER=ollama
OLLAMA_MODEL=qwen2.5:7b
OLLAMA_BASE_URL=http://localhost:11434
```

### 第二步：准备知识库文档

将学院相关资料（txt / pdf / docx / json / csv 格式）放入 `data/raw/` 目录。

项目预置了以下文档（位于 `data/raw/`）：

- 网络空间安全学院简介.md
- 网络空间安全学院领导.md
- 网络空间安全学院组织机构.md
- 网络空间安全学院教师名录.md
- 网络空间安全学院专业介绍（完整版）.md
- 网络空间安全学院教务办事指南.md
- 网络空间安全学院发展历程.md

### 第三步：一键启动

```bash
python run.py
```

启动后访问：

| 服务 | 地址 | 说明 |
|------|------|------|
| 后端 API | http://localhost:8000/docs | FastAPI 自动生成 API 文档 |
| 管理员控制台 | http://localhost:8501 | 管理员后台管理（首次自动创建 admin/admin123） |
| 用户聊天系统 | http://localhost:8502 | 用户对话界面（需要注册） |

### 分步启动（可选）

```bash
# 仅启动后端
python run.py --backend

# 仅启动管理员控制台
python run.py --admin

# 仅启动用户聊天系统
python run.py --user
```

## 四、配置文件说明

项目配置文件为项目根目录下的 `.env` 文件。完整配置项如下：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|

| `DEEPSEEK_BASE_URL` | `https://api.deepseek.com` | DeepSeek API 地址 |

| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama 服务地址 |
| `EMBEDDING_PROVIDER` | `huggingface` | 嵌入模型提供商 |
| `HF_EMBEDDING_MODEL` | `BAAI/bge-large-zh-v1.5` | HuggingFace 嵌入模型 |

| `CHROMA_PERSIST_DIR` | `data/chroma_db` | 向量数据库持久化目录 |
| `SQLITE_DB_PATH` | `data/chatbot.db` | SQLite 数据库路径 |
| `KNOWLEDGE_DIR` | `data/raw` | 知识库文档目录 |
| `RETRIEVAL_TOP_K` | `6` | 检索返回的最相关文档块数量 |
| `RETRIEVAL_SCORE_THRESHOLD` | `0.3` | 检索相关度阈值 |
| `MAX_CONTEXT_MESSAGES` | `50` | 上下文消息最大数量 |
| `MAX_AGENT_ITERATIONS` | `5` | Agent 最大迭代次数 |
| `KB_OVERVIEW` | — | 知识库概述（用于 LLM 理解知识范围） |

### 核心技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| 后端框架 | FastAPI | RESTful API 服务（端口 8000） |
| 前端框架 | Streamlit | 管理员控制台（8501）和用户聊天（8502） |
| LLM 框架 | LangChain + LangGraph | LLM 调用 + RAG 流程编排 |
| 向量数据库 | ChromaDB | 文档向量存储与相似度检索 |
| 关系数据库 | SQLite | 用户、会话、消息、审计日志持久化 |
| 嵌入模型 | BAAI/bge-large-zh-v1.5 | 中文文本向量化（HuggingFace） |
| 认证 | JWT（HS256） | API 身份认证令牌 |

### LangGraph 问答流程

```
用户输入 → [路由节点]
                ├── 命中敏感词 → [拦截节点] → 返回安全提示 → 结束
                └── 正常输入 → [检索节点] → ChromaDB 向量检索
                                        ↓
                               [LLM 生成] → 流式输出 → 结束
```

## 六、使用指南 — 管理员端

### 6.1 登录

访问 http://localhost:8501

**默认管理员账号：**

| 用户名 | 密码 |
|--------|------|
| `admin` | `admin123` |

> 首次启动系统自动创建管理员账号，请及时修改密码。

### 6.2 模型配置（Tab 1）

![模型配置界面]

**操作步骤：**

1. **选择模型提供商**：下拉选择 DeepSeek 或 Ollama
2. **填写配置**：
   - DeepSeek：API Key（必填）、API 地址（可选）
   - Ollama：服务地址（默认 `http://localhost:11434`）
3. **拉取模型列表**（可选）：点击"拉取模型列表"获取可用模型
4. **输入模型名称**：自由输入要使用的模型名
5. **保存配置**：点击"保存配置"按钮

**支持的模型提供商：**

| 提供商 | 类型 | 优点 | 缺点 |
|--------|------|------|------|
| DeepSeek | 云端 API | 无需本地 GPU，速度快，模型强 | 需要 API Key 和网络 |
| Ollama | 本地部署 | 完全离线，数据不外传 | 需要本地 GPU，速度较慢 |

### 6.3 用户管理（Tab 2）

**功能列表：**

- **查看用户**：列表显示所有用户名、角色（管理员/普通用户）
- **添加用户**：输入用户名和密码，点击"添加用户"
- **删除用户**：点击 🗑 按钮删除普通用户（不能删除自己）
- **重置密码**：点击 🔑 按钮将用户密码重置为 `123456`（不能重置管理员）

**用户角色说明：**

| 角色 | 权限 |
|------|------|
| 管理员 | 所有功能：模型配置、用户管理、知识库管理、系统控制、审计日志查看 |
| 普通用户 | 仅可使用聊天系统，提问和查看自己的会话 |

### 6.4 知识库管理（Tab 3）

#### 知识库状态

顶部展示 4 个指标卡片：

| 指标 | 说明 |
|------|------|
| 文档数量 | 知识库中文件总数 |
| 向量块数 | 文档切割后生成的向量片段数 |
| 最后构建 | 向量索引最后一次构建时间 |
| 索引状态 | 绿色"已同步"或黄色"需要重建"警告 |

#### 添加文档

提供三种添加方式：

**方式一：上传文件**
- 支持格式：txt / md / pdf / docx / json / csv
- 点击上传按钮选择文件，自动解析

**方式二：从 URL 获取**
- 输入网页 URL，系统自动抓取内容保存为 txt
- 可自定义文件名（可选）

**方式三：手动编写**
- 输入文件名（需以 `.txt` 结尾）
- 在文本框中编写内容
- 点击保存

#### 管理文档

- **预览**：👁️ 按钮展开文档内容预览
- **删除**：🗑 按钮弹出确认对话框，确认后删除文档及对应的向量块

#### 重建向量索引

当文档发生变动后，系统会显示"需要重建索引"警告。点击"重建向量索引"按钮，后台异步重建索引，可在状态栏查看进度。

> 注意：新添加或删除的文档需要重建索引后才能被检索到。

### 6.5 启动服务（Tab 4）

| 状态 | 操作 |
|------|------|
| 绿色"知识库已就绪" | 系统正常运行，可直接使用用户聊天系统 |
| 橙色"服务未启动" | 点击"启动服务"按钮预热知识库 |
| 蓝色"预热中" | 等待 1-2 分钟，点击"刷新"按钮查看进度 |

### 6.6 审计日志（Tab 5）

请参考[第九节：审计日志](#九审计日志)。

## 八、知识库管理

### 8.1 支持的文档格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| 纯文本 | `.txt` | UTF-8 编码文本文件 |
| Markdown | `.md` | 同上处理 |
| PDF | `.pdf` | 自动提取文本内容 |
| Word | `.docx` | 自动提取文本内容 |
| JSON | `.json` | 支持 `content` / `text` 字段自动识别 |
| CSV | `.csv` | 支持 `content` / `text` 列，其余列转为元数据 |

### 8.2 知识库存储位置

```
data/
├── raw/              ← 知识库源文档目录
│   ├── 网络空间安全学院简介.md
│   ├── 网络空间安全学院领导.md
│   ├── 网络空间安全学院教师名录.md
│   └── ...
├── chroma_db/        ← 向量数据库持久化目录
│   ├── chroma.sqlite3
│   └── ...
└── chatbot.db        ← 关系数据库（用户/会话/消息/审计日志）
```

### 8.3 检索配置

可在 `.env` 中调整检索参数：

```ini
RETRIEVAL_TOP_K=6               # 每次检索返回的文档块数（默认6）
RETRIEVAL_SCORE_THRESHOLD=0.3   # 相关度阈值，低于此值的结果被过滤
```

### 8.4 知识库更新流程

```
管理员上传文档 → 自动标记 stale → 手动重建索引 → 新内容可被检索
```

添加或删除文档后，知识库不会自动重建，需要在管理员控制台手动点击"重建向量索引"。

## 九、审计日志

| 操作类型 | 触发场景 |
|----------|----------|
| `login` | 用户登录（成功/失败均记录） |

| `change_password` | 用户修改密码 |
| `reset_password` | 管理员重置用户密码 |
| `create_user` | 管理员创建用户 |

| `create_doc` | 手动编写文档 |
| `fetch_url` | 从 URL 抓取内容 |

### 9.3 每条日志包含的信息

| 字段 | 说明 |
|------|------|

   - **资源类型**：按认证/用户/文档/配置/系统筛选

## 十、常见问题

### Q1：启动后无法访问服务？

**可能原因：**

- 端口被占用 → 终止旧进程：`Get-Process python | Stop-Process -Force`（Windows）
- `.env` 未正确配置 → 检查 API Key 等必填项
- 依赖未安装完整 → 重新执行 `pip install -r requirements.txt`

### Q2：Ollama 没有回复或回复中断？

**解决方法：**

1. 确认 Ollama 服务已启动：`ollama serve`
2. 确认模型已下载：`ollama list`
3. 检查管理员控制台中 Ollama 服务地址是否正确（默认 `http://localhost:11434`）
4. 系统已配置 `keep_alive=30m`，如果长时间不提问仍可能被卸载，稍等片刻自动重载

### Q3：Ollama 无法拉取模型列表？

**解决方法：**

1. 确认 Ollama 服务正在运行
2. 检查管理员控制台中"服务地址"是否与 Ollama 实际地址一致
3. 确认防火墙未阻止 11434 端口

### Q4：添加文档后搜索不到？

**解决方法：**

1. 文档添加后系统自动标记为 stale（需要重建）
2. 在管理员控制台 → 知识库管理 → 点击"重建向量索引"
3. 等待索引重建完成（状态变为绿色"已同步"）
4. 再进行搜索

### Q5：会话历史消失了？

**可能原因：**

- 登录了不同账号 → 切换回原账号
- 会话被删除 → 历史会话不可恢复
- 数据库损坏 → 检查 `data/chatbot.db` 是否存在

### Q6：如何更改模型提供商或模型？

**方法一（通过管理员控制台）：**
1. 登录管理员控制台
2. Tab 1 "模型配置" → 选择提供商 → 填写配置 → 保存

**方法二（直接修改 .env 文件）：**
1. 修改 `LLM_PROVIDER` 和对应配置
2. 重启后端

### Q7：向量索引重建很慢？

重建速度取决于：

- 文档数量与大小
- 嵌入模型运行速度（GPU > CPU）
- 系统硬件性能

首次构建因需要下载嵌入模型可能较慢（约 1-5 分钟）。

### Q8：如何查看后端日志？

后端日志输出在启动终端的 stdout，包含 INFO 级别以上的日志信息：
