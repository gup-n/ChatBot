# 项目文件说明与维护指南

本文档说明当前仓库中每个项目文件的职责、主要入口，以及通常需要修改它的场景。缓存、数据库、向量库和 `.env` 等运行时文件不纳入版本控制，因此不逐一列出。

## 根目录

| 文件 | 作用 | 何时修改 |
| --- | --- | --- |
| `README.md` | 安装、配置、运行与 Docker 使用说明。 | 部署方式或用户操作变化时。 |
| `.env.example` | 所有环境变量的无密钥模板，包含 LLM/Embedding 的 JSON 配置示例。 | 新增可配置项时。 |
| `pyproject.toml` | Python 项目元数据和直接运行时依赖。 | 代码新增或移除直接依赖时。 |
| `uv.lock` | `uv` 的确定性依赖锁文件。 | `pyproject.toml` 依赖变化后通过 `uv lock` 更新。 |
| `run.py` | 本地同时启动 FastAPI、管理员端和用户端的进程管理入口。 | 端口、启动参数或服务数量变化时。 |
| `Dockerfile` | 三个服务共用的容器镜像定义。 | Python 版本、依赖安装或镜像构建流程变化时。 |
| `docker-compose.yml` | 后端、管理员端、用户端的容器编排和数据卷。 | 容器网络、端口、挂载或环境变量变化时。 |
| `项目需求.md` | 需求设计总纲与功能范围。 | 需求基线调整时。 |
| `系统使用说明书.md` | 面向使用者的操作说明。 | 页面流程或业务功能变化时。 |

## 后端 `backend/`

| --- | --- |
| `__init__.py` | Python 包标记。 |
| `config.py` | 读取和写入 `.env`；分别维护 `LLM_SETTINGS` 和 `EMBEDDING_SETTINGS`，并提供安全与目录配置。 |
| `main.py` | FastAPI 应用、生命周期、数据库初始化、CORS 和向量库预热入口。 |
| `database.py` | SQLite 建表、用户、会话、消息和审计日志的数据库访问层。 |
| `auth.py` | 密码哈希、JWT 签发/校验和可选的初始管理员创建。 |
| `audit.py` | 非阻塞审计记录与客户端 IP 提取。 |
| `security.py` | 框架无关的外部 URL 安全校验，阻止 SSRF 访问本机和私有网络。 |
| `schemas.py` | 所有 API 请求/响应的 Pydantic 数据模型和输入长度约束。 |
| `routes/__init__.py` | 路由包标记。 |
| `routes/auth.py` | 注册、登录、改密、重置密码，以及当前用户/管理员鉴权依赖。 |
| `routes/chat.py` | 会话 CRUD、消息读取、RAG 调用和 SSE 流式回答。 |
| `routes/admin.py` | 管理端模型配置、模型列表、用户、文档、索引和审计 API；文档接口支持批量/文件夹上传、数量与大小限制。Embedding 设置变更会标记索引过期。 |
| `models/__init__.py` | 模型包标记。 |
| `models/llm.py` | LLM 工厂：支持 OpenAI 兼容的 `ChatOpenAI` 与 Ollama 原生流式聊天接口。 |
| `models/embedding.py` | Embedding 工厂：支持 HuggingFace 本地模型、OpenAI 兼容 Embedding API 和 Ollama `/api/embed`。 |
| `rag/__init__.py` | RAG 包标记。 |
| `rag/types.py` | RAG 共用数据格式；`RetrievalHit` 统一表示检索文本、来源、距离和元数据。 |
| `rag/loaders.py` | 数据载入层：txt/md/pdf/docx/json/csv 统一转换为带 `source`、`file_type` 等标准元数据的 `Document`。 |
| `rag/splitter.py` | 数据切分层：按中文自然边界切块，并为每块补充 `chunk_index`。 |
| `rag/ingestion.py` | 数据入库编排层：支持全量重建，以及新文件的“载入 → 切分 → 增量写入”。 |
| `rag/retrieval.py` | 数据检索层：阈值过滤并返回统一的 `RetrievalHit` JSON 格式。 |
| `rag/vector_store.py` | Chroma 存储层：连接、全量替换、按 `source` 增量替换 chunk、原始相似度查询、过期标记和文档块删除。 |
| `rag/graph.py` | LangGraph 的敏感内容路由和知识库检索节点。 |

## 模型配置规则

1. LLM 与 Embedding 完全独立，可使用不同服务商和模型。
2. `openai_compatible` 可连接 OpenAI、DeepSeek、OneAPI、vLLM 等兼容 `/v1` 协议的服务；填写 API Key、Base URL 和模型名。
3. `ollama` 使用本地服务地址，LLM 走 `/api/chat`，Embedding 走 `/api/embed`。
4. `huggingface` 仅用于 Embedding，模型名称例如 `BAAI/bge-large-zh-v1.5`。
5. 改动 Embedding 模型或服务商后，旧向量维度或语义空间不再可靠，必须在管理端执行“重建向量索引”。

## 环境变量配置规则

模型只使用 `LLM_PROVIDER`、`LLM_SETTINGS`、`EMBEDDING_PROVIDER`、`EMBEDDING_SETTINGS` 四项；其中 provider 选择当前启用项，两个 JSON 保存各提供商的模型、地址与密钥。旧版 `DEEPSEEK_*`、`OLLAMA_MODEL`、`HF_EMBEDDING_MODEL` 等单项模型变量已不再被读取，应从实际 `.env` 移除。

切片策略使用 `CHUNK_SIZE`、`CHUNK_OVERLAP`、`CHUNK_SEPARATORS`。最后一项是 JSON 字符串数组，默认按段落、换行、中文句号和逗号逐级切分。管理员端“知识库管理 → 切片策略”可安全修改这三项：保存会写入实际 `.env`、标记索引待重建；`.env.example` 始终保留默认值。配置不合法时，程序会拒绝保存并给出明确错误。

运行调优项包括：`LLM_TEMPERATURE`、`OLLAMA_NUM_CTX`、`OLLAMA_KEEP_ALIVE`、`LLM_STREAM_TIMEOUT_SECONDS`、`EMBEDDING_TIMEOUT_SECONDS`、`MODEL_LIST_TIMEOUT_SECONDS`、`URL_FETCH_TIMEOUT_SECONDS`、`URL_FETCH_CONNECT_TIMEOUT_SECONDS`、`JWT_TOKEN_EXPIRE_HOURS`、`AUDIT_LOG_MAX_PAGE_SIZE` 和 `HTTP_CLIENT_TIMEOUT_SECONDS`。检索策略由 `RETRIEVAL_MODE` 控制，默认 `hybrid` 融合向量与 BM25 关键词召回，`vector` 则只使用向量检索。本地启动主机与端口使用 `APP_HOST`、`API_PORT`、`ADMIN_PORT`、`USER_PORT` 配置。

## RAG 数据管道规则

统一数据流是 `loaders.Document → splitter.Chunk → vector_store.Chroma → retrieval.RetrievalHit`。所有载入后的文档至少包含 `source` 和 `file_type`；切分后的每块额外包含 `chunk_index`；检索结果始终包含 `content`、`source`、`distance` 与 `metadata`。上传、URL 获取和手动编写会按 `source` 删除旧块后写入新块，无需全量重建；只有 Embedding/切片策略变更，或索引修复时才全量重建。新增文件格式、调整分块策略或替换向量库时，应只修改对应单层模块，避免跨层调用。
