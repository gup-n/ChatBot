# 宁波工程学院网安学院问答机器人 · 需求设计总纲

---

## 一、背景介绍

选取任意的 LLM（DeepSeek、千问、GPT、Claude 等）作为基座，开发针对宁波工程学院网安学院的问答智能体。界面要求类似 DeepSeek，可以在左侧显示历史对话记录，也可以删除对话。用户可以提出问题，智能体将做出解答，但不能超过它的能力范围。

**样例问题：**
- 网安学院的院长是谁？
- 网安学院有哪几个专业？
- 负责学生竞赛的老师是哪位？
- XX 老师的办公室在哪里？
- ……

建议使用 Python 或 Java 进行开发，骨架使用 LangChain 或 LangGraph，需要配置内部知识库以支持长久记忆，知识库中将包含网安学院的各类详细信息；还需要使用 RAG 避免幻觉。

---

## 二、系统背景（已扩充）

随着人工智能与大语言模型（LLM）的快速发展，基于检索增强生成（RAG, Retrieval-Augmented Generation）的智能问答系统逐渐成为高校信息服务的重要方向。本实验旨在基于 DeepSeek 大语言模型，开发一个面向宁波工程学院网安学院的问答智能体系统。

该系统通过构建学院专属知识库，将院系基本信息（如学院领导、专业设置、教师信息、办公地点、竞赛指导教师等）进行结构化存储，并结合 LangChain 或 LangGraph 框架实现检索增强生成，从而减少大模型"幻觉"，提升回答的准确性与可靠性。

系统前端采用类似 DeepSeek 的对话式界面，支持历史对话记录展示与删除，用户可进行自然语言提问，系统基于知识库进行检索并生成回答，同时限制其回答范围在学院知识域内。

**使用的技术：** LangChain、LangGraph、Streamlit、FastAPI、ChromaDB、SQLite、uv

## 三、功能需求

| 编号 | 需求分类 | 需求描述 | 对应实现 |
|------|----------|----------|----------|
| F-01 | LLM 多模型接入 | 支持 DeepSeek API 和 Ollama 本地模型作为 LLM 基座，通过 provider 注册机制可插拔扩展 | [backend/models/llm.py](backend/models/llm.py) |
| F-02 | LLM 模型配置管理 | 管理员可在控制台切换模型提供商（DeepSeek/Ollama）、填写 API Key、设置 API 地址、拉取并选择模型 | [frontend/admin/app.py](frontend/admin/app.py) Tab 1 |
| F-03 | 用户注册 | 新用户可以自助注册普通用户账号（用户名+密码，基础校验） | [backend/routes/auth.py](backend/routes/auth.py) `POST /register` |
| F-04 | 用户登录 | 用户凭用户名和密码登录，获取 JWT 令牌 | [backend/routes/auth.py](backend/routes/auth.py) `POST /login` |
| F-05 | 修改密码 | 用户可在侧边栏修改自己的密码 | [backend/routes/auth.py](backend/routes/auth.py) `POST /change-password` |
| F-06 | 管理员重置密码 | 管理员可将任意普通用户的密码重置为 123456 | [backend/routes/auth.py](backend/routes/auth.py) `POST /reset-password` |
| F-07 | 管理员用户管理 | 管理员可在控制台查看用户列表、添加用户、删除用户 | [backend/routes/admin.py](backend/routes/admin.py) |
| F-08 | 会话管理 | 支持新建、删除、列出历史会话，会话自动以用户首条提问命名 | [backend/routes/chat.py](backend/routes/chat.py) |
| F-09 | 多轮对话 | 基于 LangGraph 的 RAG 增强对话，支持在会话上下文中连续提问 | [backend/routes/chat.py](backend/routes/chat.py) `POST /send` |
| F-10 | 流式输出（SSE） | 使用 Server-Sent Events 流式返回 LLM 回答，前端逐 token 显示 | [backend/routes/chat.py](backend/routes/chat.py) `_event_stream()` |
| F-11 | 知识库文档管理 | 管理员可上传文件（txt/md/pdf/docx/json/csv）、手动编写文本、从 URL 抓取内容、预览文档、删除文档 | [backend/routes/admin.py](backend/routes/admin.py) 知识库端点 |
| F-12 | 向量索引管理 | 文档变更后自动标记 stale，管理员可触发向量索引重建 | [backend/rag/vector_store.py](backend/rag/vector_store.py) |
| F-13 | 敏感内容拦截 | 基于关键词（色情、赌博、毒品等 13 类）检测用户输入，命中后返回固定安全提示 | [backend/rag/graph.py](backend/rag/graph.py) `_check_sensitive()` |
| F-14 | 知识库外问题拦截 | 非学院知识域的问题，LLM 被提示诚实拒绝回答 | [backend/routes/chat.py](backend/routes/chat.py) system prompt |
| F-15 | 管理员控制台 | 独立的 Streamlit 管理端（端口 8501），包含模型配置、用户管理、知识库管理、服务启动、审计日志 5 个 Tab | [frontend/admin/app.py](frontend/admin/app.py) |
| F-16 | 用户聊天界面 | 独立的 Streamlit 用户端（端口 8502），左侧历史会话侧边栏，右侧对话区域，类似 DeepSeek 界面 | [frontend/user/app.py](frontend/user/app.py) |
| F-17 | 服务健康检查 | 提供 `/api/health` 端点检查后端运行状态、预热情况、配置就绪状态 | [backend/main.py](backend/main.py) |
| F-18 | 后台自动预热 | 启动时异步加载嵌入模型与向量库，用户无需等待即可使用 | [backend/main.py](backend/main.py) `_do_warmup()` |
| F-19 | 系统审计日志 | 记录所有用户操作（登录/注册/文档管理/配置修改等），支持按用户/操作类型/资源类型筛选和分页查看 | [backend/audit.py](backend/audit.py) |

## 四、性能需求

| 编号 | 需求描述 | 对应实现 |
|------|----------|----------|
| P-01 | **异步预热**：系统启动后后台线程异步加载嵌入模型和构建向量库，不阻塞 API 响应 | [backend/main.py](backend/main.py) `start_warmup()` |
| P-02 | **流式响应**：LLM 回答以 SSE 流式传输，首 token 延迟低至模型首 token 时间，用户无需等待完整生成 | [backend/routes/chat.py](backend/routes/chat.py) `_event_stream()` |
| P-03 | **并发安全**：向量库单例使用 `threading.Lock` 保护，避免多线程竞争 | [backend/rag/vector_store.py](backend/rag/vector_store.py) |
| P-04 | **向量检索效率**：基于 ChromaDB 的相似度搜索，单次检索毫秒级响应，默认返回 Top-6 结果 | [backend/rag/vector_store.py](backend/rag/vector_store.py) |
| P-05 | **重建异步化**：向量索引重建在后台线程执行，前端可轮询状态接口查看进度 | [backend/routes/admin.py](backend/routes/admin.py) `POST /rebuild` |
| P-06 | **数据库连接池**：SQLite 采用 WAL 模式 + 每次操作独立连接，读写并发性能良好 | [backend/database.py](backend/database.py) |

## 五、安全需求

| 编号 | 安全分类 | 需求描述 | 对应实现 |
|------|----------|----------|----------|
| S-01 | **系统审计** | 所有用户操作均记录审计日志，包括操作时间、操作用户、操作类型、资源、IP 地址、成功/失败状态；审计日志不可篡改（仅追加） | [backend/audit.py](backend/audit.py)、[backend/database.py](backend/database.py) `audit_logs` 表 |
| S-02 | **身份认证** | 基于 JWT（HS256）的 Token 认证，24 小时过期，每次请求需携带 Bearer Token | [backend/auth.py](backend/auth.py)、[backend/routes/auth.py](backend/routes/auth.py) `security = HTTPBearer` |
| S-03 | **密码安全** | 密码使用 werkzeug `generate_password_hash` 哈希存储（pbkdf2:sha256），不存明文 | [backend/auth.py](backend/auth.py) |
| S-04 | **访问控制（RBAC）** | 普通用户/管理员两级角色；管理员专属端点使用 `_require_admin` 依赖注入保护；用户仅能访问自己的会话和消息 | [backend/routes/admin.py](backend/routes/admin.py)、[backend/routes/auth.py](backend/routes/auth.py) `get_current_user()` |
| S-05 | **SQL 注入防护** | 所有数据库查询均使用参数化查询（? 占位符），杜绝拼接 SQL | [backend/database.py](backend/database.py) 全局 |
| S-06 | **路径穿越防护** | 文档读取和删除操作使用 `Path.resolve().is_relative_to()` 校验，防止 `../../` 路径穿越攻击 | [backend/rag/documents.py](backend/rag/documents.py) `read_document_content()`、[backend/routes/admin.py](backend/routes/admin.py) `delete_document()` |
| S-07 | **文件名安全** | 所有用户提供的文件名使用 `Path(name).name` 剥离目录部分，防止隐藏路径 | [backend/routes/admin.py](backend/routes/admin.py) 上传/创建/删除 |
| S-08 | **敏感内容过滤** | 13 类敏感关键词检测，命中后直接拦截并返回固定安全提示，不进入 LLM 生成 | [backend/rag/graph.py](backend/rag/graph.py) |
| S-09 | **CORS 安全** | FastAPI 配置 CORS 中间件，当前允许所有源（开发阶段） | [backend/main.py](backend/main.py) |
| S-10 | **审计容错** | 审计日志记录失败不影响主业务流程（try-except 静默捕获） | [backend/audit.py](backend/audit.py) |
| S-11 | **防御深度** | 管理员删除文档时同时清理 ChromaDB 中对应的向量块，避免残留数据 | [backend/rag/vector_store.py](backend/rag/vector_store.py) `delete_document_chunks()` |
| S-12 | **操作验证** | 管理员删除用户时校验"不能删除自己"；重置密码限制不能重置管理员密码 | [backend/routes/admin.py](backend/routes/admin.py)、[backend/routes/auth.py](backend/routes/auth.py) |

| 维度 | 最终决策 |
|------|----------|
| 项目定位 | 面向了解网安学院的用户，提供多轮对话问答 |
| 核心技术 | LangChain + LangGraph + RAG |
| 交互模式 | 聊天机器人（Chatbot），类似 DeepSeek 界面 |
| 界面框架 | Streamlit 双应用（管理员端:8501 + 用户端:8502） |
| 后端框架 | FastAPI（端口 8000） |
| 数据库 | SQLite（用户/会话/消息/审计日志） + ChromaDB（向量存储） |
| LLM 接入 | 可插拔 provider 注册：DeepSeek API / Ollama 本地 |
| 知识库格式 | 支持 txt/md/pdf/docx/json/csv |
| 嵌入模型 | HuggingFace BAAI/bge-large-zh-v1.5 |
| 记忆策略 | 会话内无限记忆，由 API 上下文窗口承载 |
| 知识库外问题 | LLM 被提示诚实拒绝回答 |
| 敏感内容 | 关键词拦截 + 固定安全提示 |
| Agent 图结构 | 用户输入 → 路由(敏感检测) → (拦截/检索) → LLM 生成 → 结束 |
| RAG 策略 | 文档切割 → 向量嵌入 → 相似度检索(默认Top-6) → 附来源信息 |
| 流式输出 | SSE 流式 |
| 安全特性 | 系统审计、JWT 认证、RBAC、密码哈希、参数化查询、路径防护 |
