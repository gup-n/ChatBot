# 宁波工程学院网安学院问答机器人 · 需求设计总纲

| 维度             | 最终决策                                                                                                 |
| ---------------- | -------------------------------------------------------------------------------------------------------- |
| 项目定位         | 面向了解网安学院的用户，提供多轮对话问答                                                                 |
| 核心技术         | LangChain + LangGraph + RAG                                                                              |
| 交互模式         | 聊天机器人（Chatbot），类似 DeepSeek 界面                                                                |
| 界面框架         | Streamlit 一体化（前后端不分离，快速交付）                                                               |
| 数据库           | SQLite，存储会话和消息记录                                                                               |
| 记忆策略         | 会话内无限记忆，由 API 上下文窗口承载                                                                    |
| 知识库外问题     | 强制走固定道歉回复（A 格式）                                                                             |
| 指代不清         | 强制追问澄清，不允许猜测                                                                                 |
| 敏感内容         | 强制安全拦截，固定提示                                                                                   |
| Agent 自主范围   | 何时检索、何时用历史直接回答、最终回答生成                                                               |
| RAG 策略         | 基础 RAG（文档切割→向量嵌入→相似度检索→附来源摘要）                                                      |
| 流式输出         | 需要（SSE 流式）                                                                                         |
| 会话管理         | 支持新建/删除/导出会话，显示历史会话列表                                                                 |
| 模型接入         | 自定义，支持 API 或 Ollama 本地模型，需统一转成 LangChain 兼容接口                                       |
| 最终数据格式     | 统一为便于处理的格式（待拿到数据后确定，预留 TextLoader/JSONLoader）                                     |
| LangGraph 图结构 | 用户输入 → 强制路由节点 → (正常) Agent节点(工具调用) / (违规/指代不清/知识库外) 固定回复/追问节点 → 结束 |
| Agent 工具       | 知识库检索工具（含 RAG）、直接回答工具（或 LLM 本体回答）                                                |
| 状态字段         | messages, pending_clarification, session_id                                                              |
